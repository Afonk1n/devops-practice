## Linux & Bash — мини-шпаргалка по тому, что мы уже сделали

### 1. Навигация и работа с файлами

```bash
cd ~
```
- **`cd`** — change directory, смена директории.
- **`~`** — домашняя директория текущего пользователя.

```bash
pwd
```
- **`pwd`** — print working directory, показать полный путь к текущей директории.

```bash
mkdir devops-linux-test
cd devops-linux-test
```
- **`mkdir`** — make directory, создать папку.
- **`cd devops-linux-test`** — перейти в созданную папку.

```bash
touch notes.txt
```
- **`touch`** — создать пустой файл или обновить время последнего изменения, если файл уже существует.

```bash
ls -la
```
- **`ls`** — список файлов и папок.
- **`-l`** — показать подробную информацию (права, владелец, размер, дата).
- **`-a`** — показать скрытые файлы (те, что начинаются с `.`).

### 2. Проверка свободного места

```bash
df -h
```
- **`df`** — disk free, показывает использование дисков/файловых систем.
- **`-h`** — human readable, человекочитаемый формат (К, М, Г вместо байтов).

Команда:

```bash
df -h | head
```
- **`|`** — pipe (конвейер), передаёт вывод одной команды на вход другой.
- **`head`** — показывает первые 10 строк ввода (по умолчанию), удобно, чтобы не смотреть огромный список.

### 3. Наш Bash-скрипт `disk_info.sh`

Скрипт мы создавали через конструкцию:

```bash
cat > disk_info.sh << 'EOF'
#!/usr/bin/env bash
echo "Дата и время:"
date
echo
echo "Свободное место на дисках:"
df -h
EOF
```

Разбор:
- **`cat > disk_info.sh`** — команда `cat` читает данные из stdin (того, что ты печатаешь в терминал) и записывает в файл `disk_info.sh` (перенаправление `>`).
- **`<< 'EOF'`** — это **here-document (heredoc)**:
  - говорит shell-у: «читай всё, что я буду писать ниже, пока не встретишь строку `EOF`, и отправь это как ввод в `cat`»;
  - `'EOF'` в кавычках значит: не интерпретировать переменные и спецсимволы внутри (всё идёт как есть).
- Строка `EOF` в конце — **маркер окончания** here-doc (должна быть в начале строки, без лишних пробелов).

Содержимое скрипта:

```bash
#!/usr/bin/env bash
```
- это **shebang** — указывает, каким интерпретатором запускать скрипт;
- `/usr/bin/env bash` — найти `bash` через `env` (более кросс-платформенно, чем жёсткий путь `/bin/bash`).

```bash
echo "Дата и время:"
date
```
- **`echo`** — вывести строку.
- **`date`** — вывести текущие дату и время.

```bash
echo
```
- `echo` без аргументов — вывести пустую строку (просто перенос строки для красоты).

```bash
echo "Свободное место на дисках:"
df -h
```
- текстовое описание + сама команда `df -h` (см. выше).

После создания скрипта:

```bash
chmod +x disk_info.sh
```
- **`chmod`** — change mode, изменить права.
- **`+x`** — добавить флаг «исполнимый» (execute) к файлу.
- после этого файл можно запускать как программу: `./disk_info.sh`.

Запуск:

```bash
./disk_info.sh
```
- **`./`** — запуск файла из текущей директории (по умолчанию shell не ищет программы в `.` по соображениям безопасности).
